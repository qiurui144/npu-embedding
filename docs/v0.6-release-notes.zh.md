# Attune v0.6.0 发布说明

_发布日期：2026-04-28_

## 一句话概括

Attune v0.6.0 是 **「记忆 + 浏览 + 隐私 + benchmark」** 大版本：我们让 Attune 学会记住你反复回到的对话、悄悄从你真正读过的网页里学习、对每一条引用都更诚实，并且在通用（rust-book / cs-notes）和法律（lawcontrol）双赛道完整跑过端到端 RAG benchmark — **三个赛道全部达到 PRO 阈值，两个赛道 MRR 满分 1.00**。

## 新功能亮点

- **情景记忆固化（W1 / A1）**：反复出现的对话线程会被压缩成紧凑的「情景」，通过意图而不是关键词召回。资源治理（每线程 token 上限 + 全局内存上限）防止长会话把固化器跑飞。
- **高参与度浏览捕获（W3 / G1, G2, G5）**：开启一次后，Attune 只静默记住你**真正停留、滚动、回访**的页面，而不是你随手点开的所有标签。双层防护（`HARD_BLACKLIST` 银行/医疗黑名单 + 你自己的白名单）+ 强制阻断隐身窗口，敏感会话不进库。暂停、白名单编辑、一键「清除所有浏览数据」都直接放在扩展弹窗里。
- **更强、更诚实的 RAG（W2 / J1, J3, J5）**：每个 chunk 现在带路径前缀（`docs/foo.md > §2 > ¶3`），引用落到段落而不只是文档。新增显式召回阈值 + 二次检索，首轮置信度不够时自动追加；强约束 prompt 让 LLM 在不确定时直接说「不知道」，而不是瞎编。
- **可点击引用（W2 / B1）**：聊天中的每条引用都是真正的深链 — 一键在 Reader 面板打开原文 chunk。后端 deep-link API 已交付，UI 收口在 0.6.1。
- **Web 搜索缓存（W3 / C1）**：重复的 web 查询命中本地 30 天加密缓存（AES-GCM，按 vault 隔离）。同一个查询第二次 = 零 token 成本，零额外往返。
- **Breadcrumb 端到端透传（W3 / F2）**：检索流水线全程携带 breadcrumb 元数据并加密落盘，每个回答都能追到「哪个文件、哪一节、哪一段」 — 而且这张地图本身在磁盘上也是密文。
- **解析器 Golden Set 基线（W3 / K2）**：5 个 fixture 的金标语料 + 测试夹具守住解析器，质量回归 > 5% 自动触发人工评审。

- **隐私分级 + PII 脱敏（Phase A.5）**：三层隐私模型
  - **L0 🔒** — 文件级标记，chunk 永不出网（强制本地 LLM）
  - **L1 默认** — 12 类格式化 PII 检测（身份证 ISO 7064 校验 / 手机 / 邮箱 / API key 等）+ 可逆 `[KIND_N]` placeholder + 出网审计日志（CSV 可导出供合规审查）
  - **L3** — LLM 语义脱敏（计划高端硬件 + K3 NAS 才启用）
  - 行业 plugin（law-pro / medical-pro）在 plugin.yaml 声明额外 PII 模式（如案号、病历号），PluginRegistry 启动时自动聚合。

- **跨域污染防御（F-Pro）**：多 corpus 共享 vault 出现同语种跨域窜段（中文法律 query 拉出 cs-notes Java/算法内容）。四阶段修复：
  1. `items.corpus_domain` 元数据字段（legal/tech/medical/patent/general）
  2. chunk 文本注入领域前缀（`[领域: legal] ...`）— embedding 自动按领域聚集
  3. 跨域 penalty 0.4：query domain ≠ doc domain 时降权
  4. 关键词 query intent 检测 — 零 LLM 调用，亚毫秒
  效果：法律 Hit@10 0.60 → 0.80，Rust MRR 副得 0.87 → 1.00。

- **证据流端到端**：chat citation 现在含真实 `breadcrumb`（chunk 路径）、`chunk_offset_start/end`（Reader 跳转锚点）、`confidence`（1-5，从 LLM 强约束 marker 解析）。`routes/chat.rs` JSON marshalling 中四处数据丢失 bug 已修复。

- **Embedding backend 环境变量（`ATTUNE_EMBEDDING_BACKEND`）**：默认 ORT (Xenova/bge-m3 量化) 自包含发版；`=ollama` 切换 Ollama bge-m3 F16 + GPU（benchmark / Pro 部署用，~3.6× 提速 + 全精度）。

- **LLM 模型环境变量（`ATTUNE_CHAT_MODEL`）**：PREFERRED_MODELS 扩展加 `qwen3:8b/4b/1.7b` / `qwen3.5:35b-a3b` / `deepseek-r1:32b/14b/8b`，现代推理模型开箱即用。

- **BAAI bge-reranker-base ONNX 默认 reranker**：替代 Xenova 量化版（在某些中文长文档上触发 `Expand node invalid shape` ONNX 错误，导致 reranker 永久退化到 RRF 排序）。`ATTUNE_RERANKER_MODEL=xenova-quantized` 仍保留作为降级选项。

- **Worker 队列饥饿 fix**：`embed_queue` worker 不再因 `task_type='classify'` 任务永久 cycling（dev / bench 默认 classifier 未加载场景）。`dequeue_embeddings` 现在过滤 `WHERE task_type='embed'`。Store::open 启动时复位 stuck-processing 任务。

## 不兼容变更与迁移

- **Chrome 扩展会重新弹权限**：扩展 manifest 现在申请 `<all_urls>` 以支持浏览捕获。升级后首次启动 Chrome 会弹标准授权对话框 — 同意一次就好。（拒绝也行，捕获会保持关闭。）
- **Vault 自动迁移**：Store::open 启动时跑三个幂等迁移：
  - `chunk_breadcrumbs.breadcrumb_json`（明文）→ `breadcrumb_enc`（加密）— 老数据丢弃，indexer 下次扫描重建
  - `embed_queue` 加 `task_type` 列（默认 'embed'）
  - `items` 加 `privacy_tier`（默认 'L1'）+ `corpus_domain`（默认 'general'）
  - `bound_dirs` 加 `corpus_domain` 列
- **隐身模式硬阻断**：扩展现在拒绝在 Chrome 隐身窗口加载，这是有意为之。要捕获请用普通窗口。
- **Chat response 字段扩展**：`/api/v1/chat` JSON 现在含 `confidence: u8`（1-5，无 marker 时默认 3），citations 含 `breadcrumb` / `chunk_offset_start` / `chunk_offset_end`。已有字段不变。老客户端忽略多余字段。
- **`/api/v1/index/bind` 接受 `corpus_domain`**：可选字段，默认 `'general'`。设为 `'legal'` / `'tech'` / `'medical'` / `'patent'` 以启用跨域 penalty。

## 已知限制（坦白说）

宁愿先发出来 + 告诉你哪里粗糙，也不藏。

- **Breadcrumb 偏移是近似值**：F2 携带的是 sidecar 累计字符数，不是严格的原文 offset。引用能落到正确 chunk，但可能差几个字符。真正的精确对齐在 v0.7（J2）。
- **Web 缓存清理 UI 在 v0.6.1 上**：后端 `DELETE /api/v1/web_search_cache` 已就位；Settings UI 按钮排到 v0.6.1（W4-002）。
- **L3 LLM 脱敏排到 v0.7**：Phase A.5.6 — 需要 Tier T3+/T4+/K3 硬件 + chinese-roberta-NER ONNX 集成（~300MB）。trait 框架在 `attune-core::pii`，下游接线延期。
- **Settings → Privacy UI 排到 v0.6.1**：后端 `/api/v1/privacy/tier` + 文件级 🔒 toggle endpoint 已就绪；UI 表面在 v0.6.1（Phase A.5.5 frontend）。
- **不支持 macOS**：本版本仅 Windows（P0）+ Linux（P1）。macOS 暂不投入。
- **LLM 默认走远端 token**：Attune 本地捆绑 embedding / ASR / OCR，但 LLM 默认走你自己的 API key（或手动装 Ollama 跑本地 LLM）。`ATTUNE_CHAT_MODEL=<model_name>` 可以让高级用户绕过自动探测。
- **Domain-hash pepper 是编译期常量**：浏览域名哈希在 v0.6 用编译期常量 pepper；v0.7 升级为从 vault salt 派生，隔离更强。
- **`serde_yaml` 上游已 deprecated**：用户无感，迁移排到 v0.7（W4）。
- **122 个 route 仍用 `e.to_string()`**：`routes::errors` helper 已建立给新代码用；老 route 在 v0.7 渐进迁移避免向客户端泄露 VaultError 内部细节（W4-005 已部分完成 — vault.rs / search.rs / audit.rs / privacy.rs / web_search_cache.rs / auto_bookmarks.rs 已迁）。

## 升级指南

1. **先备份 vault**（任何时候都别省，`~/.attune/vault.db` + `~/.attune/index/`）
2. 从 release 页面下载 v0.6.0（Win MSI / Linux deb / Linux AppImage）
3. 直接覆盖安装上一版本。首次启动自动跑 schema 迁移（约 1-3 秒）
4. 打开 Chrome → 重载 Attune 扩展 → 想用浏览捕获就同意新权限弹窗，不想用就拒绝
5. （可选）打开扩展弹窗 → **Privacy** tab → 检查默认 `HARD_BLACKLIST`，加入自己的白名单或直接暂停捕获

Vault 解锁流程不变。Chat、搜索、引用在迁移过程中和迁移后都应正常工作。如有异常，请查看 `~/.attune/logs/migration.log` 并附在 issue 里。
