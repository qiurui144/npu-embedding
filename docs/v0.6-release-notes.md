# Attune v0.6.0 — Release Notes

_Released: 2026-04-28_

## Headline

Attune v0.6.0 is the **"memory + browse + privacy + benchmark"** release. We taught Attune how to remember the conversations you keep coming back to, how to quietly learn from the pages you actually read, and how to stay honest about the citations it gives you. Web search now keeps a 30-day encrypted cache so you stop paying for the same query twice. Every new capture path ships with an explicit opt-in switch — your browsing stays yours. And we ran end-to-end RAG benchmarks on both general (rust-book / cs-notes) and legal (lawcontrol) corpora — **all three tracks reached PRO threshold, two with perfect MRR=1.00**.

## What's new

- **Episodic memory consolidation (W1 / A1).** Repeated chat threads now condense into compact "episodes" you can recall by intent, not just keywords. Resource budgets (per-thread token caps + global memory ceiling) keep the consolidator from running away on long sessions.
- **High-engagement browse capture (W3 / G1, G2, G5).** Opt in once, and Attune will quietly remember the pages you actually dwell on, scroll through, and revisit — not every tab you flick past. A two-layer protection (`HARD_BLACKLIST` for banks/healthcare + your own whitelist) plus mandatory incognito blocking keep sensitive sessions out of the vault. Pause, whitelist edits, and one-click "clear all browse data" live right in the extension popup.
- **Stronger, more honest RAG (W2 / J1, J3, J5).** Chunks now carry a path prefix (`docs/foo.md > §2 > ¶3`) so citations land on the right paragraph, not the right document. A new explicit recall threshold + secondary retrieval pass kicks in when the first answer's confidence is too low, and the strict-grounding prompt makes the LLM say "I don't know" instead of guessing.
- **Click-through citations (W2 / B1).** Every chat citation is now a real deep-link — open the source chunk in the Reader pane with one click. Backend deep-link API is live; UI surfaces in 0.6.1.
- **Web search cache (W3 / C1).** Repeated web queries now hit a 30-day encrypted local cache (AES-GCM, vault-scoped). Same query twice = zero token cost, zero extra round-trips.
- **Breadcrumb passthrough (W3 / F2).** The retrieval pipeline now carries breadcrumb metadata end-to-end and persists it encrypted on disk, so you can trace any answer back to "which file, which section, which paragraph" — without leaking that map in plaintext.
- **Parse Golden Set baseline (W3 / K2).** A 5-fixture golden corpus + harness now guards the parser against silent regressions. Quality drift > 5% triggers a manual review.

- **Privacy tier system + PII redaction (Phase A.5).** Three-layer privacy model:
  - **L0 🔒** — per-file flag, chunk never leaves device (forced local LLM)
  - **L1 default** — 12-class regex PII redactor (id-card with ISO 7064 / phone / email / API keys / etc.) with reversible `[KIND_N]` placeholders + outbound audit log (CSV exportable for compliance)
  - **L3** — LLM-based semantic redaction (planned for high-tier hardware + K3 NAS)
  - Vertical plugins (law-pro / medical-pro) declare extra PII patterns in `plugin.yaml` (e.g. case-no, medical-record-no) — auto-aggregated by PluginRegistry.

- **Cross-domain pollution defense (F-Pro).** Multi-corpus shared vault was hit by same-language cross-domain bleed (Chinese legal queries pulling cs-notes Java/algorithm content). Four-stage fix:
  1. `items.corpus_domain` metadata field (legal/tech/medical/patent/general)
  2. Domain prefix injection in chunk text (`[领域: legal] ...`) — embeddings cluster by domain
  3. Cross-domain penalty (0.4) applied when query domain ≠ doc domain
  4. Keyword-based query intent detection — zero LLM call, sub-millisecond
  Effect: legal Hit@10 went 0.60 → 0.80, Rust MRR went 0.87 → 1.00 (side benefit).

- **Evidence flow end-to-end.** Chat citations now include real `breadcrumb` (chunk path), `chunk_offset_start/end` (Reader deep-link target), and `confidence` (1-5, parsed from LLM strict-prompt marker). Four data-loss bugs in `routes/chat.rs` JSON marshalling were closed.

- **Embedding backend env var (`ATTUNE_EMBEDDING_BACKEND`).** Default ORT (Xenova/bge-m3 quantized) for self-contained ship; `=ollama` switches to Ollama bge-m3 F16 + GPU for benchmark / Pro deployment (~3.6× faster, full precision).

- **LLM model env var (`ATTUNE_CHAT_MODEL`).** PREFERRED_MODELS extended with `qwen3:8b/4b/1.7b`, `qwen3.5:35b-a3b`, `deepseek-r1:32b/14b/8b` — modern reasoning models work out of the box now.

- **BAAI bge-reranker-base ONNX as default reranker.** Switched from Xenova quantized version (which triggered `Expand node invalid shape` on certain Chinese long documents and silently fell back to RRF order). `ATTUNE_RERANKER_MODEL=xenova-quantized` available for fallback.

- **Worker queue starvation fix.** `embed_queue` worker no longer infinite-cycles on `task_type='classify'` when classifier is unloaded (default dev/bench path). `dequeue_embeddings` now filters `WHERE task_type='embed'`. Stuck-processing tasks reset on Store::open.

## Breaking changes & migration

- **Chrome extension permission re-prompt.** The extension manifest now requests `<all_urls>` to enable browse capture. On first launch after upgrade, Chrome will show the standard permission dialog — accept it once and you're done. (You can decline; capture simply stays off.)
- **Vault auto-migration.** Three idempotent migrations run on Store::open:
  - `chunk_breadcrumbs.breadcrumb_json` (plaintext) → `breadcrumb_enc` (encrypted) — old rows dropped, indexer rebuilds on next scan
  - `embed_queue` adds `task_type` column (default 'embed')
  - `items` adds `privacy_tier` (default 'L1') + `corpus_domain` (default 'general')
  - `bound_dirs` adds `corpus_domain` column
- **Incognito mode is now hard-blocked.** The extension will refuse to load in Chrome's incognito windows, by design. Use a regular window if you want capture.
- **Chat response shape extended.** `/api/v1/chat` JSON now includes `confidence: u8` (1-5, default 3 if no marker) and citations include `breadcrumb`, `chunk_offset_start`, `chunk_offset_end` fields. Existing fields unchanged. Old clients ignore extra fields.
- **`/api/v1/index/bind` accepts `corpus_domain`.** Optional field, defaults to `'general'`. Set to `'legal'` / `'tech'` / `'medical'` / `'patent'` for cross-domain penalty to take effect.

## Known limitations

We'd rather ship and tell you the rough edges than hide them.

- **Breadcrumb offset is approximate.** F2 carries a sidecar character count, not a strict in-document offset. Citations land on the right chunk but may be off by a few chars. Real alignment lands in v0.7 (J2).
- **Web cache cleanup UI ships in v0.6.1.** Backend `DELETE /api/v1/web_search_cache` is wired; Settings UI button arrives v0.6.1 (W4-002).
- **L3 LLM-based PII redactor ships v0.7.** Phase A.5.6 — needs Tier T3+/T4+/K3 hardware + chinese-roberta-NER ONNX integration (~300MB). Trait scaffolding is in `attune-core::pii`; downstream wiring deferred.
- **Settings → Privacy UI ships in v0.6.1.** Backend `/api/v1/privacy/tier` + per-file 🔒 toggle endpoints are live; UI surface arrives v0.6.1 (Phase A.5.5 frontend).
- **macOS not supported.** Windows (P0) and Linux (P1) only this release. macOS deferred.
- **LLM is remote-token by default.** Attune ships embedding/ASR/OCR locally, but expects you to bring your own LLM API key (or install Ollama manually for local LLM). `ATTUNE_CHAT_MODEL=<model_name>` overrides auto-detect for power users.
- **Domain-hash pepper is compile-time.** Browse-domain hashing uses a build-time constant pepper for v0.6; v0.7 derives it from your vault salt for stronger isolation.
- **`serde_yaml` is upstream-deprecated.** No user-visible impact; migration scheduled for v0.7 (W4).
- **122 routes still use `e.to_string()`.** `routes::errors` helper is in place for new code; existing routes will be migrated in v0.7 to never leak VaultError internals to clients (W4-005 partial — vault.rs / search.rs / audit.rs / privacy.rs / web_search_cache.rs / auto_bookmarks.rs done).

## Upgrade guide

1. **Back up your vault** (always — `~/.attune/vault.db` and `~/.attune/index/`).
2. Download v0.6.0 from the releases page (Win MSI / Linux deb / Linux AppImage).
3. Install over the previous version. First launch will run the schema migration automatically (~1-3 seconds).
4. Open Chrome → reload the Attune extension → accept the new permission prompt if you want browse capture, or leave it disabled.
5. (Optional) Open the extension popup → **Privacy** tab → review the default `HARD_BLACKLIST` and add your own domains to whitelist or pause capture entirely.

Vault unlock works exactly as before. Chat, search, and citations should all continue to function during/after migration. If anything looks wrong, check `~/.attune/logs/migration.log` and file an issue with the log attached.
