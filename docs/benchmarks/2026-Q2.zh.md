## 方法学

### 1. 确定性指标（W4 J6，v0.6 落地）

对 `rust/tests/golden/queries.json` 每个 query：

- **Hit@K**：top-K 中至少 1 个 `acceptable_hits[]` ID 命中 → 1，否则 0
- **Recall@K**：`|top-K 中命中| / |acceptable_hits|`
- **MRR (Mean Reciprocal Rank)**：第一个命中的 `1/rank`（无命中 = 0）

纯 Rust 计算（无 LLM 依赖，完全可复现）。`rag_quality_benchmark.rs` 单元测试用固定输入
验证指标实现。

### 2. LLM-judged 指标（W5 J6，v0.6.1+ 计划）

按 [RAGAS framework](https://github.com/explodinggradients/ragas)：

- **Faithfulness**：答案的论断是否被检索 context 支撑？
- **Answer Relevancy**：答案是否回答了用户问题？
- **Context Precision**：相关 chunk 是否排在 context 前面？
- **Context Recall**：context 是否覆盖了 ground truth 答案？

需要靠谱 LLM judge（一般 GPT-4 级）。等 W5 C4 multi-LLM router 稳定后再 v0.6.1 接。

### 3. 真 corpus harness

mock corpus 测试在 `rag_quality_benchmark.rs` 每 PR 跑。真 corpus benchmark 用
`#[ignore]` 挡住默认 cargo test，手工触发：

```bash
cd rust
cargo test --release -p attune-core --test rag_quality_benchmark -- --ignored
```

避免每 PR 都烧 30 分钟 CI 跑完整 index + query。

## 为什么 v0.6 不带最终数字也发

按 W4 plan J6，**方法学 + framework + 可复现性**才是 v0.6 deliverable。Baseline 数字在
真 corpus 跑数后 v0.6.0 GA 落地 — 不阻塞 v0.6.0-rc.1。

这是有意：先锁方法学（哪些 query / 哪个 corpus / 哪个 K / 哪些指标），**再**看数字 —
防止数字出来后改目标的经典坑（"goalpost shifting after the data comes in"）。

## 如何复现

1. checkout Attune v0.6.0-rc.1 标签
2. 按 `_corpus_pins` 固定版本拉 corpus：
   ```bash
   bash scripts/download-corpora.sh
   ```
3. 跑 benchmark：
   ```bash
   cd rust
   cargo run --release --bin attune-server &  # 后台
   sleep 5
   curl -X POST http://localhost:18900/api/v1/index/bind -d '{"path": "../tests/corpora/rust-book"}'
   # 等 indexing 完成（看 /api/v1/status）
   cargo test --release -p attune-core --test rag_quality_benchmark -- --ignored --nocapture
   ```
4. 对比公开 baseline。

## 诚实的局限

- **v0.6 没有 LLM-judged 指标**（Faithfulness/AR/CP/CR）。确定性指标覆盖检索质量，
  抓不住生成答案的幻觉。补上它是 v0.6.1 W5 J6 优先级
- **每 corpus 单语言** — 混合语言 query（如中英混合法律术语）需要单独场景，v0.7 加
- **没有延迟 benchmark** — 质量优先，延迟在 v0.7 加 PromptIQL 缓存层时再做
